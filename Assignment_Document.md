Assignment Description
This is a Group assignment. The objective of this assignment is to design and implement
microservices architecture for real world business domains that have identified. Through
this assignment,
• Students will have the basic idea of microservice architecture.
• Students will understand the role the API gateway plays in microservice
architecture.
• Students will have exposure to the actual development environment and the
hands-on experience on tools and API documentation.
Assignment Task - Implement microservice backend for the minimal viable product
 (MVP) identified for any real-world scenario
For example, if your business domain is school, then you can select sub domains like
"Student => Student API" and "Course => Course API"
Or if your business domain is shopping cart, then you can select sub domains like
"Product => Product API" and "Customer => Customer API"

Expectation Criteria:
1. Define and elaborate microservices you have selected (microservice per each
member of the group).
2. Elaborate how you can avoid having multiple ports with the gateway you
implemented.
3. Solution should have proper folder structure with microservices and an API Gateway.
4. Include screen captures of your native swagger URL outcomes and the same when
you access them using the API gateway.
Assignment Approach:
1. Group Assignment.
2. Need to submit a slide deck by covering the entire scenario. Include the screen
captures of the final solution.
3. The final solution should have microservices you identified and the API gateway.
4. Your final solution should not have any build breaks or any run time errors. Both the
end points should be able to be accessed by using the API gateway as well as
separately.
Deliverables:
1. A slide deck elaborating the domain you've chosen.
2. A brief description of microservices you've identified.
3. Illustrate the need of the API gateway.
4. A working solution with microservices and API gateway. Take screen captures of the
final solution and enclose to the slide deck
5. The slide deck should contain screen captures of the end points when they are being
called directly as well as via API gateway.
