## 🧪 Product Service Example

### main.py

```python
from fastapi import FastAPI

app = FastAPI()

products = []

@app.get("/products")
def get_products():
    return products

@app.post("/products")
def add_product(product: dict):
    products.append(product)
    return {"message": "Product added", "data": product}
```

### Run:

```bash
uvicorn main:app --port 8001 --reload
```

## 🌐 STEP 2: API Gateway (IMPORTANT)

### Purpose:

* Avoid multiple ports
* Single entry point
* Route requests to services

---

### api-gateway/main.py

```python
from fastapi import FastAPI
import requests

app = FastAPI()

# Product Service
@app.get("/products")
def get_products():
    return requests.get("http://localhost:8001/products").json()

@app.post("/products")
def add_product(product: dict):
    return requests.post("http://localhost:8001/products", json=product).json()


# Customer Service
@app.get("/customers")
def get_customers():
    return requests.get("http://localhost:8002/customers").json()


# Order Service
@app.get("/orders")
def get_orders():
    return requests.get("http://localhost:8003/orders").json()


# Payment Service
@app.post("/pay")
def make_payment(data: dict):
    return requests.post("http://localhost:8004/pay", json=data).json()


# Inventory Service
@app.get("/inventory")
def get_inventory():
    return requests.get("http://localhost:8005/inventory").json()
```

---

### Run Gateway:

```bash
uvicorn main:app --port 8000 --reload
```

## ❓ Key Concept: Why API Gateway?

Without API Gateway:

* Users must access multiple ports

With API Gateway:

* Single entry point (Port 8000)
* Better security
* Centralized routing
